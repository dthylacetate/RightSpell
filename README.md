## 一、 产品定位  
“拼好队” 是一款专为校园音乐人设计的垂直社交小程序。它解决了传统校园社群（如微信群、共享文档）信息碎片化、更新快、难检索的痛点，为乐手组队、技术交流提供了一个高效的“信息集散地”。

## 二、 用户画像   
**发起人** ： 正在组建乐队或准备排演曲目的组织者，急需特定乐器位置（如键盘、主唱）。  

**野生乐手** ： 拥有乐器技能，希望寻找志同道合的队友或被乐队收编。

**爱好者**： 关注校园演出资讯，热衷于在讨论区交流的校友。

## 三、 核心功能设计 (Core Logic)  
### 1.双向招募市场 (Dual-Direction Marketplace)
**标签化区分**： 为了降低用户的认知成本，系统采用了“橙/蓝”双色标签体系。

**招募乐手 (橙色)**： 代表乐队/项目寻找人才，侧重于“招聘”属性。

**个人求组队 (蓝色)**： 代表乐手个人展示技能，侧重于“简历”属性。

**信息原子化**： 帖子强制包含“目标乐器”和“音乐风格”，确保信息在列表页即能实现初步筛选。

### 2. 自动化实名名片系统 (Identity System)
**同步机制**： 平台通过关联 users 集合，实现了发帖与评论时的用户信息自动抓取。  
用户只需在“我的”页面完善一次资料，全站即可实现“实名化”社交，增强了校园环境下的信任感。

## 四、 技术架构 (Technical Architecture)
### 1. 技术栈  
**前端**： 微信小程序原生框架 (WXML/WXSS/JS)。

**后端**： 微信云开发 (WeChat Cloud Development)。

**云数据库**： 非关系型数据库 (JSON)，处理帖子、留言、用户信息的关联。

**云存储**： 托管用户头像及招募海报，使用 cloud:// 协议确保持久访问。

**云函数**： 处理高权限逻辑（如安全检测）。

## 五、 安全与合规性 (Security & Compliance)
这是 1.0 版本最核心的隐形功能：

**内容安检门神**： 接入了微信 openapi.security.msgSecCheck 2.0 接口。

**全链路过滤**： 无论是发帖标题、正文还是微小的留言，都会通过云函数进行“内容体检”。

**逻辑拦截**： 采用“先校验，后入库”的闯关模式，从根源上杜绝了违法违规信息进入数据库，确保小程序能顺利通过微信官方审核。

## 六、 1.0 开发总结与 2.0 规划 (Roadmap)
### 1. 1.0 已解决的问题
完成了从 0 到 1 的全栈闭环。

解决了跨页面用户信息同步的同步性问题。
